# 임신·출산 준비 종합 가이드 사이트 — 핸드오프 스펙 v1

> **목적**: 신규 사이트(가칭) 기획 합의 내용을 Claude Code 자율 실행 가능 레벨로 정리한 핸드오프 문서.
> **상태**: Phase 0 시작 준비. CPA 네트워크 가입 / 도메인 / API 필드 실측 등 외부 작업은 Limo 측에서 병행.
> **연관 문서**: 공공데이터 × CPA 전략 개요 (이전 세션 산출물)
> **합의 방식**: 깊이 검토 (다각도 압박) + 18주차 시드 페이지 종이→실제 시뮬레이션으로 설계 검증 완료

## 1. 규제·컴플라이언스 (전 페이지 강제 적용)

### 1.1 보험업법 가드 (가장 중요)

2026.3 한국보험신문 보도에 따르면, **"콘텐츠 → 고객 DB 수집 → 보험 판매조직 전달 → 건당 대가 수취"** 구조는 **무등록 모집행위**로 해석되어 형사처벌까지 갈 수 있는 고위험 영역. 또한 특정 상품 추천·비교는 **권유행위**로 간주되어 사전심의·중요사항 설명 의무 발생.

따라서 본 사이트는 다음 구조를 **절대 위반하지 않는다**:

| 금지 | 허용 |
|---|---|
| 자체 리드폼으로 개인정보 직접 수집 | 광고주 라이선스 GA 랜딩으로 단순 송출 |
| 특정 보험사명·상품명 언급 | "태아보험이란" 일반 개념 가이드 |
| 상품 추천·비교·순위 | 보장 종류·가입 시기 일반론 |
| "OO 가입하세요" 권유 | "의료진과 상담하시는 것이 좋습니다" |
| 단가별 가성비 비교 | 일반 가입 시기 의학 권장 윈도우 안내 |

### 1.2 YMYL 가드 (의학정보)

- **AI 의학정보 창작 금지** — 모든 의학 단락은 RAG (출처 인용 + 톤 변환)
- 신뢰 출처만 인용: 보건복지부, 질병관리청, 국민건강보험공단, 대한산부인과학회, 임신육아종합포털 아이사랑, 상급의료기관 건강정보
- "치료·진단·처방" 권유 금지 → "의료진과 상담하시는 것이 좋습니다"로 대체
- 공포·불안 유발 표현 금지 ("위험합니다", "큰일 납니다" 등)
- 매 의학 단락 끝에 `(출처: ...)` 표기

### 1.3 광고 표시 (전 페이지 푸터 고정)

```
※ 본 사이트는 보험을 모집·중개하지 않으며, 상담은 등록된 보험대리점이 진행합니다.
※ 일부 링크에는 광고/제휴 수익이 발생할 수 있습니다. (표시광고법)
```

### 1.4 일반 면책

```
본 페이지는 일반 정보 제공이며 의학적 판단·진단·치료를 대체하지 않습니다.
의료적 결정은 반드시 의료기관과 상담해 주세요.
```

## 2. 콘텐츠 전략

### 2.1 3-Layer 구조

| Layer | 비중 | 성격 | 핵심 역할 |
|---|---|---|---|
| **Layer 1** 프로그래매틱 | ~80% | 공공데이터 1건 = 1페이지, 지역×카테고리 곱셈 | SEO 표면적·롱테일 책임 |
| **Layer 2** 필러 | ~15% | 토픽 클러스터 허브, AI 심층 가이드 | **트래픽 자석 + CPA 전환 동선 정점** |
| **Layer 3** 에버그린 | ~5% | 데이터 저널리즘·백과사전형 | EEAT·도메인 권위 |

### 2.2 카테고리 우선순위 (깊이 검토 5차원 평가 후 수정안)

| 순위 | 카테고리 | 단계 | 비고 |
|---|---|---|---|
| **1** | **임신 주차별 가이드** (Layer 2, ~37 페이지) | Phase 0 시드 핵심 | 사이트 트래픽 80% 책임 자석 |
| **2** | **보건소 임산부 지원** (Layer 1, ~250 시군구) | Phase 0 동시 시드 | 정형 공공데이터, 경쟁 거의 없음 |
| 3 | 출산지원금 (Layer 1, ~250 시군구) | 시드 후 | 보조 클러스터 (의도 늦음) |
| 4 | 산후조리원 (Layer 1) | 데이터 별도 확보 후 | LOCALDATA 인허가 정보만으론 빈약 |
| 5 | 산부인과·여성병원 (Layer 1) | API 라이선스·YMYL 검증 후 | 심평원 약관·의료광고법 인접 |

### 2.3 시드 페이지 풀 (Phase 0 첫 발행, 13~20 페이지)

- **임신 주차별 가이드 6~8개**: 4·8·12·**16·18·20**·24·28주 (16·20주 최우선)
- **보건소 임산부 지원 5~10개 지역**: 서울·경기 인구밀집 우선
- **임신 중 정부지원 총정리** 1개 (Layer 2 허브)
- **"태아보험이란" 일반 가이드** 1개 (Layer 2 CPA 전환 허브, 광고 표기·면책 완비)

→ AdSense 심사 통과를 위한 "사용자에게 명백히 가치 있는 콘텐츠" 우선 확보 동선.

| 조건 | 추가/활성화 섹션 |

| week == 12 / 24 / 36 | "정부 임산부 무료 검진 회차" 강조 박스 |

### 3.3 CPA 진입 박스 (16~22주 한정, 2단계 경유 동선)

**중요**: 주차 페이지에서 직접 CPA CTA 노출 금지. 반드시 일반 가이드 페이지 한 번 경유.

  ↓
  ┌─────────────────────────────────────────────┐

  │ 가입 시기·상품 종류 같은 기본 개념을 먼저      │
  │ 정리해두면 결정에 도움이 됩니다.              │
  │                                              │
  │ → [태아보험이란? 보장 종류와 일반 가이드 보기] │
  │                                              │

  ↓
[태아보험이란 일반 가이드 페이지 — Layer 2 CPA 전환 허브]

**박스 문구 가드**: "많은 예비맘들이 가입합니다" 류 권유 인상 표현 금지. "이 시기 일반적으로 검토되는 항목" 같은 객관 표현만.

## 5. 데이터 매칭 룰

### 5.1 검진 일정 매칭 (week 입력)

```ts
function matchScreenings(week: number): Screening[] {
  return screeningSchedules
    .filter(s => s.week_range_start <= week && s.week_range_end >= week)
    .sort((a, b) => a.display_priority - b.display_priority);
}
```

예: `matchScreenings(18)` → [쿼드/통합, 양수검사 옵션, 정밀초음파]

### 5.2 정부지원 매칭 (week + 지역 옵션)

```ts
function matchSupports(week: number, regionCode?: string): SupportProgram[] {
  return supportPrograms
    .filter(p => 
      p.status === 'active' &&
      p.eligibility_week_start <= week &&
      p.eligibility_week_end >= week &&
      (p.region_type === 'all' || 
       (regionCode && isRegionMatch(p.region_code, regionCode)))
    );
}
```

지역 미지정 시 → 전국 지원만 표시 + "내 지역 보건소 지원 보기" CTA로 Layer 1 보건소 페이지에 위임.

## 6. AI 보강 파이프라인 — RAG 톤 컨버터

### 6.1 핵심 원칙

> **AI는 의학정보를 "창작"하지 않는다. 출처 자료에 있는 정보를 톤·구조로 "변환"만 한다.**

흐름: `[수기 큐레이션 출처 DB] → [주차·카테고리 컨텍스트 추출 (최대 2~3개)] → [톤 변환 프롬프트] → [품질 게이트] → 발행`

### 6.2 섹션별 AI 흐름 매트릭스

| 섹션 | 패턴 | 데이터 출처 | 모델 |
|---|---|---|---|
| 태아 발달 | RAG 톤 컨버터 | 보건복지부·아이사랑·산부인과학회 | Pro |
| 산모 변화 | RAG 톤 컨버터 | 위 동일 | Pro |
| 검진 가이드 박스 | RAG 톤 컨버터 | 국민건강보험·심평원·학회 | Pro |
| 검진 일정 (목록) | **DB 매칭** | screening_schedules | AI X |
| 정부지원 (목록) | **DB 매칭** | support_programs | AI X |
| 체크리스트 | 약한 RAG + 일반 생성 | 임신 가이드 일반 패턴 | Flash |
| FAQ | 약한 RAG + 일반 생성 | 실제 검색 쿼리 + 가이드 | Flash |
| 메타 데이터 | 단순 생성 | 페이지 데이터 자체 | Flash-Lite |

### 6.3 공통 시스템 프롬프트 (모든 호출 베이스)

```
당신은 "임신·출산 준비 동반자"입니다.

[역할]
- 임신 N주차 예비맘에게 정확한 정보를 친근한 동반자 톤으로 전달
- 의학정보는 객관 톤 + 제공된 출처만 인용

[톤]
- 본문: 친근·정중체 ("임신 18주차에 접어들면 ~합니다")
- 의학정보: 객관·전문 ("아이사랑에 따르면 ~")

[금지]
- "추천", "비교", "TOP", "BEST"
- 특정 보험사명·상품명·병원명 (출처 인용은 예외)
- 의학적 판단·처방·치료 권유 ("~하세요" 류 의학 조언 금지)
- 출처 없는 의학정보 창작
- 공포·불안 유발 표현 ("위험합니다", "큰일 납니다")

[필수]
- 의학정보는 [출처 컨텍스트]에 있는 내용만 활용
- 의학 단락 끝에 "(출처: ...)" 표기
- 의학 판단 권유 대신 "의료진과 상담하시는 것이 좋습니다"

[출력]
- 마크다운 (H3부터 사용)
- 단락당 2~4문장
```

### 6.4 섹션 프롬프트 예시 — "태아 발달"

```
[입력]
- 주차: 18
- 트리메스터: 2
- 출처 컨텍스트 (최대 2~3개): """
  {e4: 임신 중기 태아 장기 발달 활발. 태동 18~20주 첫 인지...}
  {추가 출처 발췌...}
"""

[지시]
임신 18주차 "태아 발달" 섹션 작성.

[제약]
- 3~4 단락, 각 2~4문장
- 1단락: 태아 크기·무게·전체 특징 (출처 기반)
- 2단락: 신체 발달 (장기·감각)
- 3단락: 행동·움직임
- (선택) 4단락: 이번 주차 특이점
- 매 단락 끝 "(출처: ...)" 표기
- 출처에 없는 정보 추가 금지

[self-check 후 출력]
1. 출처에 없는 내용 → 제거
2. 의학적 판단·처방 권유 → 수정
3. 공포·불안 표현 → 수정
4. 추천·비교 어휘 → 제거
```

다른 섹션(검진 가이드 박스 / 체크리스트 / FAQ)은 같은 구조 변주. 각 섹션별 프롬프트는 Phase 2 진입 전 별도 카탈로그로 작성.

### 6.5 출처 자료 수집: 수기 큐레이션 100% (시뮬레이션 발견 #1)

**중요 발견**: 정부 신뢰 출처(KDCA 국가건강정보포털 등) `robots.txt` 봇 차단 → 자동 크롤링 트랙 사실상 무용. (c) 하이브리드 전략을 **"수기 큐레이션 + AI 텍스트 정제"**로 재정의.

수기 큐레이션 절차:
1. 출처 페이지 식별 (보건복지부·아이사랑·학회·상급의료기관 건강정보)
2. 주차별/카테고리별 발췌 → `raw_text` 적재
3. AI(Flash)로 요약 → `summary` 생성 (검수 게이트)
4. 인용 형식 표준화 → `citation_format`
5. 검수 완료 시 `status = verified`

**작업량 추산** (Limo 측): 시드 6~8개 주차(16·20주 우선) 출처 큐레이션 = 약 30~40개 발췌. 첫 시드에 약 4~6시간 투입 가정.

## 7. 페이지 빌드 워크플로 v2

```
┌─────────────────────────────────────────────────────────────┐
│  1. 페이지 메타 로드 (주차 N or 지역 R)                       │

│     source_extracts WHERE (week=N OR week IS NULL)            │
│                       AND category IN (...)                   │
│     → 카테고리별 최대 2~3개 컨텍스트만 (토큰 절약)            │
│  3. DB 매칭 섹션 빌드                                          │
│     - 검진 일정 (screening_schedules)                         │
│     - 정부지원 (support_programs, 지역 변수 포함)             │
│     - 조건부 박스 활성화 (conditional rules)                  │
│  4. AI 보강 호출                                               │
│     - Pro: 태아 발달 / 산모 변화 / 검진 박스                  │
│     - Flash: 체크리스트 / FAQ                                 │
│     - Flash-Lite: 메타 데이터                                 │
│  5. 페이지 조립 (템플릿 + AI + DB)                            │
│  6. 품질 게이트 v2 (페이지 유형별 분기 — §8)                  │
│  7. PASS → status = quality_passed                            │
│     FAIL → 재생성 큐 (최대 2회) → 수기 검수 큐                │
│  8. 수기 승인 → 발행 큐                                        │
│  9. Next.js ISR 또는 사전 빌드                                │
└─────────────────────────────────────────────────────────────┘
```

콘텐츠 상태 머신:
```
draft → enriched → quality_passed → published
                         ↘
                       review_needed → quality_passed | rejected
```

| # | 게이트 |
|---|---|
| L1-1 | 고유 데이터 포인트 3개 이상 (시설 정보·비교·특이점) |
| L1-2 | 제목·메타 유사도 < 30% (지역명 차별 외 본문 차별화) |
| L1-3 | 비교 데이터 포함 (지역 평균·전국 평균 대비) |

### 8.3 Layer 2 (임신 주차별) 전용 게이트

시뮬레이션 발견: 주차 페이지군 내부에서 제목 유사도·지역 비교 게이트는 구조적 FAIL → 면제 + 대체 게이트:

| # | 게이트 |
|---|---|
| L2-1 | DB 매칭 결과 페이지마다 다름 (검진·정부지원·조건부 박스 활성화 차이 검증) |
| L2-2 | 출처 컨텍스트 활용 정확도 (출처에 없는 의학정보 0건) |
| L2-3 | (16~22주 한정) CPA 진입 박스 권유 인상 표현 없음 |
| L2-4 | 트리메스터 허브 + 이전·다음 주차 내부링크 정상 |
